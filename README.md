main() создаёт конкретного строителя (ClassicBurger) и передаёт его кассиру. Cashier::assemble() вызывает методы строителя в нужной последовательности. Строитель знает, какие именно ингредиенты подставить («Говяжья котлета», «Чеддер» и т.д.). Готовый Burger* возвращается клиенту.

Легко добавлять новые позиции меню
Хотите добавить «Острый бургер»? Просто создайте новый класс SpicyBurger : public BurgerBuilder с реализацией 5 методов. Код кассы, UI, логика корзины — не меняются.
Гибкая кастомизация заказа
Пользователь хочет «двойную котлету» или «без лука»? 
- Директор может принимать флаги (extraPatty, noOnion) и условно пропускать/повторять шаги.
- Или можно создать декоратор-билдер, который оборачивает базовый и модифицирует шаги.
